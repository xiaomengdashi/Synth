# Bilibili 视频 AI 总结优化方案

## 1. 现状分析
目前系统中针对 Bilibili 视频的总结逻辑，仅仅通过 API 抓取了视频的**标题 (title)**、**简介 (description)** 和 **UP主名称 (uploader)**，然后直接将这些极其有限的元数据交给大模型（LLM）进行总结。
这就导致了总结内容非常空洞、“水”，因为 LLM 根本没有拿到视频实际讲述的**核心文稿内容**。

## 2. 业界主流的 AI 音视频总结方案
通过联网搜索与分析（如 BibiGPT、Bili2text 等知名开源/商业案例），业界针对 Bilibili 视频的 AI 总结普遍采用**“双轨制文稿提取”**策略：

1. **第一轨：提取外挂/平台 AI 字幕（首选、速度最快、成本最低）**
   - Bilibili 目前为绝大多数视频自动生成了 AI 字幕（或者 UP 主自己上传了 CC 字幕）。
   - 通过 B 站官方接口获取视频的 `cid`，再请求字幕列表接口，可以直接拿到包含时间戳和文本的完整 JSON 字幕文件。
   - 提取其中的纯文本，拼接后即可作为视频的完整文稿。
   
2. **第二轨：音频提取 + ASR 语音识别（兜底方案）**
   - 对于少数没有字幕的视频，使用 `yt-dlp` 提取纯音频流（`bestaudio`）。
   - 将音频发送给 ASR（自动语音识别）模型（如 OpenAI 的 Whisper 模型，或本地部署的 Faster-Whisper）转录为文本。

3. **大模型（LLM）长文本总结**
   - 将提取到的几千甚至上万字的**完整视频文稿**，结合标题和简介，通过特定的 Prompt 交给长上下文大模型进行分析。
   - 生成摘要、核心观点列表、思维导图等结构化内容。

## 3. 本项目 (Synth) 优化实施计划

为了快速且高效地提升 `Synth` 项目的 B 站总结效果，计划分步骤实施以下改进：

### 阶段一：接入 B 站官方字幕 API（本次核心修改）
由于 B 站目前的 AI 字幕覆盖率极高，我们将优先实现第一轨方案，这足以解决 90% 以上的视频总结质量问题。

- **步骤 1：获取视频 CID**
  - 继续使用现有的 `https://api.bilibili.com/x/web-interface/view?bvid={bvid}` 接口，不仅获取标题和简介，还要提取返回数据中的 `cid` 字段。
- **步骤 2：请求字幕接口**
  - 请求 `https://api.bilibili.com/x/player/v2?bvid={bvid}&cid={cid}`。
  - 解析返回的 JSON，查找 `data.subtitle.subtitles` 数组。
  - 如果存在字幕，获取 `subtitle_url`，并请求该 URL 下载完整的字幕 JSON 数据。
- **步骤 3：文本拼接与清理**
  - 遍历字幕 JSON 中的 `body`，提取所有的 `content` 字段并拼接为一段完整的纯文本视频文稿。
- **步骤 4：优化 LLM Prompt**
  - 将提取到的**完整文稿**注入到 Prompt 中。
  - 修改 Prompt 要求 LLM 深入阅读文稿内容，提炼核心观点、金句，并以 Markdown 结构化输出。

### 阶段二：增加 Whisper 音频转录兜底（可选扩展）
- 如果发现字幕接口未返回数据，则调用 `yt-dlp` 下载音频流。
- 使用项目中已有的 `AsyncOpenAI` 客户端，调用 `client.audio.transcriptions.create` 接口（需兼容配置中的 API Key 和 Base URL）。
- 将转录出的文本作为文稿输入给 LLM。
*(注：由于某些第三方 Base URL 可能不支持 audio 接口，此方案可作为后续配置开关项)*

## 4. 预期效果
优化后，生成的文章将不再是基于“标题和简介”的无端猜测，而是**基于视频真实讲述内容的深度提炼**，质量将发生质的飞跃，完全达到市面上主流 AI 视频总结工具的水平。
